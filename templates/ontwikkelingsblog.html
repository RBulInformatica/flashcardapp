Ontwikkelingsblog

Flashcards zijn een veelgebruikte manier om grote hoeveelheden informatie te onthouden. Er is alleen één nadeel: je kent de leerstof, maar begrijpen is iets anders. Als je het begrijpt, dan kun je het ook toepassen. Vanuit hier ontstond het idee om een flashcardwebsite te maken met AI-integratie, zodat de gebruiker het begrip kan uitleggen en het daardoor ook goed kan begrijpen. Dit is geïnspireerd op de Feynman-techniek van natuurkundige Richard Feynman, waarbij je ingewikkelde concepten heel simpel moet uitleggen.

De ontwikkeling is begonnen met het maken van een plan van aanpak. Hierbij heb ik gekozen om te beginnen met het bouwen van een website met een Flask-backend, een JavaScript-frontend en HTML voor de interface. Ook heb ik besloten om SQLite met DB Browser te gebruiken om de flashcards op te slaan in een database. Voor de AI-integratie is gekeken naar verschillende opties. Uiteindelijk heb ik gekozen voor Hugging Face, een Python-pakket voor natuurlijke taalverwerking. Eén database is gebaseerd op een online gevonden begrippenlijst over het onderwerp 'immuunsystemen' voor het vak biologie.

Na het programmeren ben ik begonnen met het testen van de site. Op dat moment was er nog geen AI-integratie en werkte nog niet alles. Daarna heb ik functionele flashcards toegevoegd en een functie gemaakt om alle kaarten te bekijken via een volledig menu.

Nadat ik tevreden was over de basisfunctionaliteit, ben ik begonnen met de AI-integratie. Ik heb een extra optie toegevoegd om naast normaal oefenen ook met AI te oefenen. De gebruiker krijgt dan een score op basis van semantische gelijkenis. Als deze score hoog genoeg is, kan je naar de volgende kaart. Tijdens het testen bleek dat het model soms correcte antwoorden afkeurde als ze in een andere vorm gegeven werden. Aangezien dit model niet zo goed getraind is als ChatGPT, Copilot of andere modellen, moet de gebruiker nog steeds een redelijk vergelijkbaar antwoord geven. De beste manier om dit op te lossen zou zijn door een ander taalverwerkingsmodel zoals ChatGPT te gebruiken, maar daaraan zijn kosten verbonden.

Na de basis van de website te hebben gemaakt, wilde ik hiervan een werkende en bezoekbare versie maken. Eerst heb ik gekeken naar Firebase van Google. Na het installeren van Firebase en Node.js kwam ik erachter dat ik daarvoor de hele structuur van mijn project moest aanpassen. Na wat verder zoeken ontdekte ik dat ik Render kon gebruiken om een website te hosten zonder de structuur te wijzigen. Hiervoor moest ik een GitHub-account aanmaken en Git installeren. Vervolgens heb ik een repository gemaakt vanuit de bestanden op mijn computer. Ook heb ik render.yaml en requirements.txt toegevoegd. Deze heb ik verbonden met Render en gedeployed. Daarbij liep ik tegen het probleem aan "No open ports detected", wat ik heb opgelost door Waitress toe te voegen.

Toen de website via Render werkte, ontstond er een nieuw probleem: de flashcards-database was niet verbonden met Render. Eerst heb ik geprobeerd de database handmatig aan te maken met een ander Python-script vanuit een CSV-bestand, maar dit werkte niet. Daarna probeerde ik databases te vermijden, maar ook dat lukte niet. Ik vermoed dat het opslaan van informatie niet werkt op Render. Zelf flashcards toevoegen voor de gebruiker lijkt ook niet te werken. Uiteindelijk is het gelukt door SQLalchemy te vermijden en terug te gaan naar de oude werkwijze met SQLight 3. Ook heb ik de ontwikkelingsblog toegevoegd met een apart html bestand waar naar gelinkt wordt en heb ik voor stijl een styles.css bestand gebruikt.

Op GitHub zijn alle aanpassingen te vinden. Ook is een lokale versie bijgevoegd met aanpassingen voor Render.
